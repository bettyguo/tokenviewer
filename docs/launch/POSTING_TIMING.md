# Posting timing

## Hacker News

**Recommended window:** a weekday, Tuesday–Thursday, 07:00–10:00 US Eastern
(roughly 12:00–15:00 UTC). This puts the post on the front-page cycle as the
US wakes up while Europe is still active — the widest concurrent technical
audience. Avoid Friday and the weekend (lower traffic, slower comment
velocity) and avoid late US evening (the post ages out overnight).

**Why this matters for this project:** the post lives or dies on the first
hour of upvotes and comments. Be at a keyboard for the two hours after
submitting to answer questions — the multilingual numbers will draw
methodology questions, and a fast, specific reply is worth several upvotes.

**One submission.** If it does not catch, do not repost the same day. HN
allows a second attempt after a couple of weeks if the first got little
attention; use a different title angle if so.

## X / Twitter

Post the thread the same day as the HN submission, shortly after — ideally
once the HN post has a few comments, so the thread can ride any HN traffic.
Best engagement window is roughly 09:00–11:00 or 13:00–15:00 US Eastern on a
weekday. The hero image in tweet 1 is doing the work; make sure it renders
crisply in the timeline preview.

## Blog post

Publish the blog post the same day, before the HN and X posts go out, so both
can link to it. It is the durable artifact — HN and X drive a spike, the blog
post is what gets found later.

## Bonus: ride a relevant news cycle

This launch pairs naturally with any active discussion of multilingual model
fairness, tokenizer efficiency, or a new tokenizer release. If a major model
or tokenizer ships in the days around launch, lead with the comparison against
it — "here is the new tokenizer next to the last six" is a stronger hook than
a cold launch. Watch for: a new open-weight model release, a fresh
tokenization-fairness paper, or renewed discussion of byte-level /
tokenizer-free models. Do not delay the launch indefinitely waiting for one,
but if a relevant story is live in launch week, anchor to it.

## Re-launch cadence

Every new tokenizer added is a legitimate reason to post again — "added
\<tokenizer\>, here is how it compares" — on a multi-week cadence, not more
often. This is a recurring-content project, not a one-shot launch.
